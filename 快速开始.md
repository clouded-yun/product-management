# 快速开始指南

## 快速启动步骤

### 1. 安装依赖
```bash
cd product_management_demo
npm install
```

### 2. 启动开发服务器
```bash
npm run dev
```

浏览器会自动打开 http://localhost:3000

### 3. 登录系统
- **默认管理员账号**：
  - 用户名：`admin`
  - 密码：`admin123`

### 4. 开始使用
1. 点击右上角"界面管理"创建新界面
2. 在数据界面中点击"编辑列"配置表格列
3. 点击"添加记录"开始录入数据

## 功能演示流程

### 创建第一个产品参数界面
1. 登录后，点击"界面管理"
2. 点击"创建新界面"
3. 输入界面名称：`产品参数`
4. 点击"保存"
5. 点击左侧的"产品参数"界面

### 配置表格列
1. 点击"编辑列"按钮
2. 添加以下列：
   - 列名：`product_name`，显示名称：`产品名称`，类型：`文本`
   - 列名：`price`，显示名称：`价格`，类型：`数字`
   - 列名：`description`，显示名称：`描述`，类型：`多行文本`
   - 列名：`image`，显示名称：`产品图片`，类型：`图片`
3. 点击"保存"

### 添加产品数据
1. 点击"添加记录"
2. 填写产品信息
3. 上传产品图片
4. 点击"保存"

### 导出数据
1. 点击"导出CSV"或"导出JSON"
2. 文件会自动下载

## 代码更新后如何保留数据

**重要提示**：在更新代码前，请务必先备份数据！

### 更新代码的步骤：

1. **备份数据**（更新前）
   - 登录系统（管理员账号）
   - 点击右上角"备份/恢复"按钮
   - 点击"导出备份文件"
   - 保存备份文件到安全位置

2. **更新代码**
   - 将新版本的代码覆盖旧版本
   - 重新安装依赖（如果需要）
   - 启动新版本系统

3. **恢复数据**（更新后）
   - 打开新版本系统
   - 点击"备份/恢复"按钮
   - 点击"选择备份文件"
   - 选择之前保存的备份文件
   - 等待恢复完成
   - 刷新页面或重新登录

### 备份文件说明：
- 备份文件为JSON格式
- 文件名包含导出时间戳
- 包含所有用户账号、界面配置和数据记录
- 可以安全地在不同设备间传输

## 常见问题

**Q: 数据会丢失吗？**
A: 数据存储在浏览器LocalStorage中，清除浏览器数据会丢失。更新代码前请使用"备份/恢复"功能导出备份。

**Q: 更新代码后数据会丢失吗？**
A: 不会，只要在更新前使用"备份/恢复"功能导出备份，更新后导入备份即可恢复所有数据。

**Q: 如何创建普通用户账号？**
A: 在登录页面点击"创建新账号"，选择角色为"普通用户"。

**Q: 可以导入数据吗？**
A: 可以使用"备份/恢复"功能导入完整备份。单个数据表的导入功能在后续版本中添加。

**Q: 图片大小有限制吗？**
A: 系统会自动压缩图片，但建议单张图片不超过5MB。
